Issue Report: System Hang During High I/O Operations

Issue Summary

HSD No.: 4557682314

Title: System Hang Observed During High I/O Workload

Description: During high input/output (I/O) operations, the system becomes unresponsive, requiring a hard reset to recover. This issue occurs intermittently under heavy disk and network activity.

Hardware Configuration

  • Hardware Model: HP ProLiant DL380 Gen10
  • Storage Adapter: HPE Smart Array P408i-a SR Gen10
  • Storage: 4x 1.92TB SSD, RAID 10
  • Memory: 128 GB RAM
  • CPU: Intel Xeon Silver 4216 (2x)
  • Firmware Version: 3.55.0

Software Configuration

  • Operating System: CentOS 7.9
  • Kernel Version: 3.10.0-1160.42.2.el7.x86_64
  • Storage Driver Version: hpsa 3.4.20
  • RAID Configuration: RAID 10 (4 SSDs)
  • File System: XFS
  • Network Configuration Tool: ifcfg 1.15.0

Steps to Reproduce

  1. Initiate a large file transfer between the local SSDs and a remote NFS server.
  2. Simultaneously perform disk-intensive operations such as database queries or large-scale backups.
  3. Monitor system responsiveness and I/O activity.
  4. Observe if the system becomes unresponsive during the operation.

Expected Result

The system should remain responsive, even under high I/O workloads, and all operations should complete successfully without interruption.

Actual Result

The system becomes unresponsive during heavy I/O operations and requires a hard reset to recover.
